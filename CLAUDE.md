# report-bot プロジェクト概要

## このツールの目的
通信代理店（古河店）の月次実績データ画像をAIで読み取り、
分析レポートを自動生成してGoogle Chatに投稿するツール。

## 想定ユーザー
- 古河店のマネージャー・担当者
- Streamlit CloudのURLにアクセスして使う

## ファイル構成
- app.py：StreamlitのUI（画像アップロード・データ確認・投稿）
- analyze.py：GPT-4o Visionによる画像解析・レポート生成・Google Chat投稿
- requirements.txt：Streamlit Cloud用の依存パッケージ
- .streamlit/config.toml：ダークテーマ設定
- .env：APIキー（ローカル開発用・Gitに含めない）

## 抽出する実績項目
### モバイル
新規合計、MNP合計、機変合計、UQ→au、総販合計
（目標・実績は件数、達成率・標準進捗・全国平均は%）

### LD系
auでんき、auPayカード、auPayゴールドカード、auじぶん銀行、au+1総販付帯率
（au+1総販付帯率のみ目標・実績も%で表示）

### BBC
auひかり+BL光、auひかり、BL光
（目標・実績は件数、達成率・標準進捗・全国平均は%）

## レポート出力フォーマット
Google Chat向けプレーンテキスト。以下の構造で出力する。

```
【実績報告】M/DD時点 良い点・改善点総評

▼総評 [2〜4文。全体状況・良い点・課題を記述]

--------------------------------------------------------------------------------
✅ 良い点（順調・達成圏内） ※標準進捗および全国平均を上回っている項目です。

[項目名]目標：[目標]件 / 実績：[実績]件
（達成率：[達成率] / 標準進捗 [標準進捗]）
[任意コメント]

--------------------------------------------------------------------------------
⚠️ 改善点・リカバリー目標（残りN日） ※標準進捗に対して未達の項目です。（ ）内は
本日含む残り期間（M/DD～M/DDのN日間）での1日あたり必達獲得数です。

【モバイル系】
[項目名]目標：[目標]件 / 実績：[実績]件
（達成率：[達成率] / 標準進捗 [標準進捗]）日割り目標：[X.X]件 / 日（残り[N]件）

【金融・LD系】 / 【BBC】
（同形式）
```

- Markdown記法は絶対に使わない（#, *, **, -, _ など）
- %単位の項目（au+1総販付帯率）は「件」を「%」に、日割り目標は省略
- 日割り目標はPython側で事前計算してGPTに渡す（計算ミス防止）

## 開発ルール
- APIキーは必ず.envまたはStreamlit Secrets経由で管理する
- 画像ファイルはGitに含めない（.gitignoreで除外済み）
- 店舗追加はapp.pyのSTORE_CONFIGSに追記するだけでよい
